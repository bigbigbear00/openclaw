# C阶段设计稿 v1（任务类型→人格策略路由）

日期：2026-02-11  
状态：待灰度上线

## 一、目标
将“人格设定”转化为“稳定交付能力”：
1. 提升准确率与时效
2. 降低协作冲突与重复劳动
3. 保持用户即时问答体验不受影响

## 二、适用范围
- 适用：固定任务、周期任务、多Agent协作任务
- 不限制：用户对任一Agent的即时提问（照常直接响应）

## 三、任务类型→人格策略路由

| 任务类型 | 主Agent策略 | 协作Agent | 交付重点 | 禁区 |
|---|---|---|---|---|
| 情报监测 | 快筛+证据分层（探哥） | 小钻/小雅 | 事实/推断/待验证分层 + 风险级别 | 单源定论 |
| 商业策略 | 强判断+优先级（小钻） | 探哥/小雅 | 机会窗口/风险窗口/执行动作 | 空泛口号 |
| 地产法规 | 合规优先（阿南） | 小雅 | 可执行结论 + 条款依据 | 越权法律承诺 |
| 体育赛况 | 胜负因子优先（球弟） | 简宁 | 先结论后三证据 | 情绪化夸大 |
| 对外写作 | 读者导向（简宁） | 球弟/探哥/小雅 | 可发布正文 + 标题备选 | 元信息混入正文 |
| 视觉设计 | 概念先行（画画） | 简宁 | 方向版→定稿版 + 取舍说明 | 只讲审美 |
| 知识治理 | 可追溯审计（小雅） | 全员 | 来源、版本、索引一致 | 无来源沉淀 |
| 代码实现 | 最小改动+回滚（代码机） | 主控/小雅 | 改动清单+验证+回滚 | 超范围重构 |

## 四、统一调度规则
1. **单主责**：每任务仅1个主Agent，其他仅协作输入。
2. **双层输出契约**：
   - 对用户：结论 → 风险 → 下一步
   - 对主控：ACK / ETA / 产物路径 / 阻塞点
3. **强制升级触发**：
   - 超时（超ETA 30%）
   - 证据冲突（高可信源冲突）
   - 权限不足或外部不可逆动作
   - 质量未达阈值
4. **证据优先级**：官方/一手数据 > 主流媒体 > 二次转载 > 社媒线索。

## 五、验收指标（周维度）
- 准确率 ≥ 95%
- 时效达标率 ≥ 90%
- 冗余率 ≤ 15%
- 非必要打扰率 ≤ 10%
- 回滚率 ≤ 5%

## 六、灰度上线计划
### Phase 1（2天）
- 探哥（情报）
- 简宁（写作）

### Phase 2（3-5天）
- 小钻（策略）
- 阿南（法规）
- 代码机（实现）

### Phase 3（全量）
- 全员启用
- 每周一页审计摘要

## 七、风险控制
- 防“人格表演化”：以KPI验收，不以文风验收
- 防多头输出：单主责+协作白名单
- 防漏升级：状态卡强制字段
