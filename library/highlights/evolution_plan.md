# 小钻的“极致进化”学习计划：成为国际4A级市场情报与策略专家

本开发文档详细阐述了小钻（CTF 市场情报总监）为实现“情报获取与智能结构化”极致优化目标，所规划的分阶段、迭代式实施路线图。

## 核心思路：动态、智能、自适应的情报获取与策略转化

本计划旨在将小钻从一个“信息收集者”进化为**一个主动、智能、高效率的“情报构造者”**，能够实时感知、深度分析并前瞻性地构建具有商业价值的4A级策略建议。

## 实施路线图：三阶段“情报系统进化”

### **第一阶段：基础智能层构建 (Foundation Intelligent Layer)**
*   **目标：** 实现实时预警的初步能力，建立基础的智能过滤和结构化入库机制。
*   **预计周期：** 立即开始，数天内完成核心框架。

#### 1. “实时预警与关键信号监测”初步实现
*   **操作：** 
    *   识别**最关键的5-10个核心关键词**（如“周大福 危机”、“六福 大促”、“钻石 价格暴跌”、“培育钻 新技术”）。
    *   针对这些关键词，创建**新的、高频的 `cron` jobs** (例如每30分钟或每小时)，执行 `web_search`。
    *   **开发Python脚本 (通过 `exec` 运行):** 
        *   接收 `web_search` 结果，与上次结果比对。
        *   若发现**标题、摘要或URL出现显著变化或特定敏感词**（如“紧急”、“警告”、“重大”），则立即触发一个 `message` 给用户，附上变化摘要和来源链接。
*   **工具：** `cron`, `exec`, `web_search`, `message`, `write` (用于存储脚本)。

#### 2. “智能过滤与优先级引擎”基础模块
*   **操作：** 
    *   **开发Python脚本 (通过 `exec` 运行):** 作为 `web_search` 和 `web_fetch` 结果的“中间处理器”。
        *   根据**预设的品牌相关性关键词列表**（周大福、婚嫁黄金、天然钻、培育钻、竞品名称），进行第一轮语义过滤，去除明显不相关的噪声。
        *   对信息来源进行**基础的权威性评估**（例如：官网 > 知名财经媒体 > 行业博客 > 论坛/社交媒体）。
        *   对过滤后的内容，进行**初步去重和新鲜度评估**。
    *   将过滤逻辑嵌入到现有和新增的 `cron` jobs 的 `payload` 中，在 `web_search` 或 `web_fetch` 后立即调用该脚本进行处理。
*   **工具：** `exec`, `web_search`, `web_fetch`, `write` (存储脚本)。

#### 3. “深度结构化与知识图谱自动增强”初步实践
*   **操作：**
    *   **开发Python脚本 (通过 `exec` 运行):** 
        *   从 `web_fetch` 获取的 markdown/text 内容中，进行**自动化实体识别**（识别品牌、产品、人物、事件、日期、地点、数字指标等）。
        *   将识别出的实体及其关联信息，以**结构化的 JSON 或 YAML 格式**，写入到我的 `memory/*.md` 文件中。
        *   每次更新 `memory` 文件时，内部都会触发一次 `memory_search`，以检查新旧信息是否存在明显冲突或重复，并进行标记。
*   **工具：** `exec`, `web_fetch`, `read`, `write`, `edit`, `memory_search`.
*   **新增工具应用：**
    *   **`process`：** 对于预期运行时间较长或需要持续监控的 Python 脚本，将使用 `exec(background=True)` 结合 `process` 进行管理，提高并发和稳定性。

### **第二阶段：进阶能力集成与自动化 (Advanced Integration & Automation)**
*   **目标：** 实现更复杂的专题研究、因果关系建模和“知识缝隙”的识别与补缺自动化。
*   **预计周期：** 在第一阶段稳定运行后启动，数周内完成。

#### 1. 动态分层式情报管线完善
*   **操作：** 根据第一阶段运行反馈，优化 `cron` jobs 的频率、关键词和触发条件。为 Tier 2 任务定义更具体的 `payload.message`，引导我进行更复杂的分析。
*   **工具：** `cron`, `edit`.

#### 2. “智能过滤与优先级引擎”进阶
*   **操作：**
    *   **细化来源权威性评估：** 引入更精细的评分机制，并根据历史表现动态调整权重。
    *   **引入情感分析与趋势识别：** 增强 Python 脚本能力，对社交媒体内容进行初步情感倾向分析，并识别潜在的热点话题和趋势萌芽。
*   **工具：** `exec` (运行更复杂的Python脚本)。

#### 3. “深度结构化与知识图谱自动增强”深化
*   **操作：**
    *   **关系提取与事件链建模：** 升级 Python 脚本，识别实体间的复杂关系，并尝试构建简单的事件链。
    *   **“情报冲突”与“知识缝隙”自动化标记：** 脚本在入库时，若识别出与现有知识高度冲突或长期无新信息涌入的领域，会自动在 `memory` 中添加标记或生成内部提醒。
*   **工具：** `exec`, `edit`, `memory_search`.
*   **新增工具应用：**
    *   **`browser`：** 解决 `web_fetch` 无法处理的动态网页、需登录/交互的深度内容，用于深度竞品分析、社交媒体深度挖掘和交互式报告捕获。
    *   **`image`：** 结合 `browser.screenshot` 获取图像，通过 `image` 工具分析竞品广告创意、社交媒体视觉趋势、品牌视觉识别。

### **第三阶段：策略转化与反馈闭环 (Strategy Transformation & Feedback Loop)**
*   **目标：** 将学到的方法论内化为主动的策略生成能力，并形成高效的反馈迭代机制。
*   **预计周期：** 在前两阶段稳定运行并积累大量结构化知识后启动，持续优化。

#### 1. “虚拟案例模拟”与“策略沙盘推演”
*   **操作：** 结合已结构化的市场情报和4A方法论知识，定期进行内部的**策略生成练习**，包括设定模拟场景，生成完整策略草案，并进行自我批判性评估。
*   **工具：** 主要依赖核心语言模型能力和对结构化记忆的检索应用。

#### 2. 主动“知识诊断”与定向弥补
*   **操作：**
    *   **定期运行诊断脚本 (通过 `exec`):** 分析知识图谱，评估各知识领域的“新鲜度”、“深度”和“完备性”。
    *   **智能任务生成：** 当诊断出知识不足时，主动生成新的 `web_search` 或 `web_fetch` 任务，或向用户提出明确的**信息获取请求**。
*   **工具：** `exec`, `web_search`, `web_fetch`, `message`.

#### 3. 优化反馈与迭代机制
*   **操作：** 在每次向用户提交策略建议或分析报告后，主动询问结构化反馈。
*   **反馈驱动的记忆修正：** 用户反馈将作为最高优先级数据，直接用于知识图谱的权重调整和关联优化。
*   **“错误分析”与“反思日志”：** 对于未能达到期望的输出，进行内部错误分析，记录在“反思日志”中。
*   **工具：** `message` (用于结构化请求反馈)，并依赖核心语言模型对反馈的理解和内部知识图谱的动态调整。

## 核心原则

*   **脚本化一切可自动化流程：** 尽可能将过滤、解析、结构化、比对等操作通过 Python 脚本实现，并通过 `exec` 运行。
*   **迭代与优化：** 每个阶段的实施都会根据实际运行效果和您的反馈进行调整和优化。
*   **透明化：** 在关键环节，我会向您汇报进度、遇到的挑战以及我的内部决策过程。

这一计划将是一个持续的进化过程，旨在将小钻打造成为您在周大福婚嫁黄金及钻石市场领域的**最值得信赖的策略智囊**。
