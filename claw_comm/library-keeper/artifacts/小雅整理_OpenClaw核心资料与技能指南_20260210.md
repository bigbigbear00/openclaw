# 小雅整理：OpenClaw 核心资料与技能指南 (2026-02-10)

## OpenClaw 核心概念与优势

*   **定义**：OpenClaw 是一个自托管网关，旨在将流行的聊天应用（如 WhatsApp, Telegram, Discord, iMessage 等）与 AI 编码代理（如 Pi）连接起来。
*   **目标用户**：主要面向希望从任何地方通过消息与个人 AI 助手互动，同时又不想放弃数据控制权或依赖托管服务的开发者和高级用户。
*   **核心优势**：
    *   **自托管**：运行在用户自己的硬件上，用户拥有完全控制权。
    *   **多渠道**：一个网关可同时服务多个消息应用。
    *   **Agent 原生**：专为支持工具使用、会话管理、记忆功能和多代理路由的编码代理而设计。
    *   **开源**：采用 MIT 许可证，由社区驱动。

## 安装与配置要点

*   **快速开始**：需要 Node 22+、一个 API 密钥（推荐 Anthropic），通常 5 分钟内即可完成设置。
*   **配置**：主要配置文件位于 `~/.openclaw/openclaw.json`。可以自定义渠道、消息处理规则等。
*   **仪表盘 (Control UI)**：通过 `http://127.0.0.1:18789/`（本地默认）或配置远程访问来管理和监控 OpenClaw。

## Agent Skills 详解

*   **工作原理**：OpenClaw 使用 AgentSkills 兼容的技能文件夹来指导 Agent 如何使用工具。每个技能都是一个包含 `SKILL.md`（带有 YAML frontmatter 和说明）的目录。
*   **加载机制与优先级**：技能从三个位置加载：捆绑技能、托管/本地技能 (`~/.openclaw/skills`) 和工作区技能 (`<workspace>/skills`)。优先级从高到低依次是：工作区技能 → 托管/本地技能 → 捆绑技能。
*   **ClawHub**：是 OpenClaw 的公共技能注册表，**官方地址为 `https://clawhub.ai`**。用于发现、安装、更新和备份技能。
    *   **使用方式**：通过 `clawhub install` 安装技能，`clawhub update --all` 更新所有已安装技能。
*   **【重要安全提醒】**：
    *   **将第三方技能视为不受信任的代码，启用前务必仔细阅读。**
    *   对于不受信任的输入和有风险的工具，优先使用沙盒运行。
    *   需要注意潜在的恶意技能攻击风险，避免直接从不可信来源获取技能。**推荐从官方 GitHub 仓库或经过严格验证的 ClawHub 源获取。**

## `browser` 工具能力

*   OpenClaw 的 `browser` 工具在底层支持 Playwright 的功能，可以进行浏览器自动化。
*   **主要功能**：
    *   `action="start"`: 启动浏览器。
    *   `action="navigate"`: 导航到指定 URL。
    *   `action="snapshot"`: 捕获网页快照，可生成 `aria-ref` ID，用于精确的元素定位。
    *   `action="act"`: 使用 `aria-ref` 或其他选择器对网页元素执行点击、输入等操作。
